# 3D Spheroid Validation Report

Closes #196. This report documents how the 3D spheroid simulation
infrastructure (sim-tme-3d, landed in PR #219) is validated, what it
currently predicts, and where the gap to **independent experimental
validation** sits.

## Self-consistency targets (`targets.yaml`)

Three regression-guards land with this PR:

1. **`3d_rsl3_o2_collapse_ratio`** — `hypoxic / normoxic < 0.5` for RSL3
   at λ=120, immune-off. **Catches**: regression where RSL3 stops
   collapsing in hypoxia (e.g., basal_ros decoupled from O₂).
2. **`3d_immune_sdt_dominates`** — `SDT / RSL3 immune-kills > 3` at
   λ=120, immune-on. **Catches**: immune block silently no-opping
   (DAMP-diffusion stability bug, immune_kill_probability returning 0,
   etc.).
3. **`3d_stromal_boundary_shielding`** — `stromal-on / no-stromal
   adjacent-kill < 0.7` at λ=120, immune-on. **Catches**: CAF boost
   (gsh + mufa) silently ineffective.

All three are **self-consistency** — they regression-guard the model's
own predictions, not independent experimental values. The threshold
(0.5, 3.0, 0.7) reserves a margin against current measured values
(0.016, 4.0, 0.485) so noise doesn't cause spurious failures, while
still catching catastrophic regressions.

Each target's `note` field documents the upgrade-to-`calibration` path:
which PMID-backed measurement would replace the heuristic value.

## Literature gaps — upgrade-to-calibration candidates

The following references are **already cited in the library docstrings**
but the targets above don't reach a `calibration` (independent-data)
quality bar yet. A manuscript curator should identify exact measured
values for upgrades:

| Target | Citation already in code | What's needed to upgrade |
|--------|--------------------------|--------------------------|
| `3d_rsl3_o2_collapse_ratio` | Vaupel, *Cancer Res* 1989 (`oxygen.rs` module doc) | Measured RSL3 kill rate in pimonidazole-positive (hypoxic, < 1.3% O₂) vs negative spheroid layers |
| `3d_immune_sdt_dominates` | None in code yet | Published spheroid SDT-immune-priming data (manuscript curator: Nguyen 2019 or equivalent — DAMP-density-vs-T-cell-infiltrate ratio) |
| `3d_stromal_boundary_shielding` | PMID 34373744 (`stromal.rs` module doc — CAF metabolic reprogramming) | Measured RSL3 IC50 shift in CAF-coculture spheroids vs RSL3-only |

Adding the missing `source_pmid` + measured value to the target's YAML
block is the only code change needed to promote it; the extraction
metric stays the same.

## Limitations explicit

- **Single grid size**. All current targets use 60³ (tumor radius
  540 µm). Real spheroids span 100 µm–5 mm; multi-size sweeps require
  #194's perf work + memory budget reasoning. The single grid was
  deliberately chosen at the smaller end of biologically meaningful
  spheroids; literature comparisons should weight smaller-spheroid
  measurements more heavily.
- **DAMP-density artifact in Q2**. The 4× immune ratio in 3D vs 104×
  in 2D is partly a finite-size effect — fewer tumor cells (12 k in
  3D vs 159 k in 2D) → fewer kills → fewer DAMP sources → less
  saturation of the DC-activation Michaelis-Menten. A multi-grid
  sweep would quantify how much of the ratio shift survives at
  matched in-vivo scale. Issue #194 (sim-spatial-3d perf) is
  prerequisite to running 80³+ at this depth of sweep.
- **No anti-PD-1 condition in 3D yet**. sim-tme has it; sim-tme-3d
  v0.1.0 skipped it. The Q2 "104:1 vs 4:1" framing should hold at
  anti-PD-1 too, but isn't verified. Documented as a follow-up in
  `sim-tme-3d/README.md`.
- **Pure-geometry simulation**. The model doesn't include
  3D-vasculature effects (#191) which would make 3D MORE hypoxic
  via reduced O₂ supply. The Vaupel 1989 "3D more hypoxic" observation
  reflects this biology, which the pure-geometry simulation
  intentionally separates from grid effects.
- **Cross-geometry tests inline sim-tme math**. Three library tests
  (`oxygen::tests::matched_lambda_2d_vs_3d_zone_fractions`,
  `ph::tests::matched_lambda_2d_vs_3d_acidic_fraction`,
  `stromal::tests::matched_dims_3d_boundary_fraction_exceeds_2d`)
  copy sim-tme's 2D math. If sim-tme tunes those formulas, the tests
  silently drift. Tracked in #220.

## How to re-run

```bash
# 1. Regenerate sim-tme-3d's output (from simulations/)
cd simulations
cargo run --release -p sim-tme-3d

# 2. (Optional) Refresh 2D + 3D comparison
python3 ../scripts/generate_3d_comparison_table.py

# 3. Run all calibration targets including the 3 new 3D ones
python3 calibration/calibrate.py --evaluate
```

Production run takes ~17 seconds on 8 cores. After a refresh, all 3
3D targets should show `PASS`; older simulation outputs will show
`STALE` (informational — value within tolerance, file older than
source code).
