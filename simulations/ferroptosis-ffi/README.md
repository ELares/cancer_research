# ferroptosis-ffi

C FFI bindings for the ferroptosis-core simulation engine.

Provides a C-compatible API for embedding the ferroptosis biochemistry engine in C/C++ frameworks such as PhysiCell, CompuCell3D, or Chaste.

## Building

```bash
cd simulations
cargo build --release -p ferroptosis-ffi
```

This produces `target/release/libferroptosis_ffi.dylib` (macOS) or `libferroptosis_ffi.so` (Linux). The C header `include/ferroptosis.h` is auto-generated by cbindgen during the build.

## Quick start (C)

```c
#include "ferroptosis.h"

FerroRng *rng = ferro_rng_new(42);
FerroParams params = ferro_params_default();
FerroCell cell = ferro_gen_cell(2, rng);          /* 2 = Persister */
FerroResult res = ferro_sim_cell(&cell, 1, &params, rng);  /* 1 = RSL3 */
printf("Dead: %d, LP: %.2f\n", res.dead, res.final_lp);
ferro_rng_free(rng);
```

Compile and link:
```bash
cc -o test_ffi tests/test_ffi.c -L ../target/release -lferroptosis_ffi -lm
DYLD_LIBRARY_PATH=../target/release ./test_ffi     # macOS
LD_LIBRARY_PATH=../target/release ./test_ffi        # Linux
```

## API

| Function | Description |
|----------|-------------|
| `ferro_rng_new(seed)` | Create a seeded RNG (one per thread) |
| `ferro_rng_free(rng)` | Free the RNG (no double-free) |
| `ferro_params_default()` | Default 2D biochemistry parameters |
| `ferro_params_invivo()` | In-vivo parameters (with SCD1/MUFA) |
| `ferro_gen_cell(phenotype, rng)` | Generate a cell with stochastic variation |
| `ferro_sim_cell(cell, treatment, params, rng)` | Run full 180-step simulation |

## Enum values

- **Phenotype:** 0=Glycolytic, 1=OXPHOS, 2=Persister, 3=PersisterNrf2, 4=Stromal
- **Treatment:** 0=Control, 1=RSL3, 2=SDT, 3=PDT

## Safety

- Each thread must own its own `FerroRng` (not thread-safe)
- Caller must free `FerroRng` with `ferro_rng_free` (no double-free)
- NULL cell pointer to `ferro_sim_cell` returns a safe default result (dead=false)
