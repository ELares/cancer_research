# Authoring Rules

Rules for anyone writing book content. Read this before opening a PR that touches `article/drafts/v1.md`.

## Source of Truth

`article/drafts/v1.md` is the single source of truth for all manuscript prose. `v1.tex` is auto-generated by `scripts/generate_latex.py` — never hand-edit it. `v1.pdf` is compiled from `v1.tex` by pdflatex.

**Pipeline:** `v1.md` → `generate_latex.py` → `v1.tex` → `pdflatex + bibtex` → `v1.pdf`

If Markdown and LaTeX diverge, the Markdown wins. Regenerate the LaTeX.

## Local Compilation

```bash
python3 scripts/generate_latex.py
cd article/drafts && pdflatex v1 && bibtex v1 && pdflatex v1 && pdflatex v1
```

The CI pipeline (`.github/workflows/release-pdf.yml`) runs this automatically on pushes to main that touch `article/`.

## Heading Convention

```
# Part N: Title          →  \part{Title}
## Chapter N: Title       →  \chapter{Title}
### N.N Title             →  \section{Title}
#### N.N.N Title          →  \subsection{Title}
```

- Parts use Roman numerals (I, II, III, IV, V)
- Chapters use Arabic numerals (1-11) globally, not per-Part
- Sections use chapter-relative numbering (6.1, 6.2, ...)
- The document class is `report` (not `book`) — chapters do not force recto page starts

## File Structure

The manuscript is a single file (`v1.md`). This scales to ~50,000 words (~3,000 lines). If the file exceeds 4,000 lines, revisit whether to split into per-chapter files.

**Generated content** (regenerate, don't hand-edit):
- `article/drafts/v1.tex`
- `article/drafts/v1.pdf`
- Figures in `article/figures/` (generated by scripts)

**Handwritten content** (the source of truth):
- `article/drafts/v1.md`
- Analysis documents in `analysis/`
- This file (`AUTHORING.md`) and `book-outline.md`

## Narrative Inflation Guardrails

These rules prevent the expansion from inflating confidence beyond what the evidence supports.

1. **More prose does not upgrade confidence.** A simulation finding that was "hypothesis-generating" in the compressed article remains "hypothesis-generating" in the expanded chapter. Expanding the explanation around a result does NOT make the result more certain.

2. **Every simulation claim states its key assumptions.** The phrase "our simulations show" must be followed — in the same paragraph or a linked callout — by the model's key simplifications (e.g., steady-state fields, estimated parameters, no adaptive evolution). If the assumptions aren't stated, the claim is incomplete.

3. **"What could make this wrong" is mandatory.** Every simulation finding section must include an explicit caveats paragraph. This is not optional even when the finding looks compelling in prose form.

4. **Distinguish evidence layers.** Every factual claim should be traceable to its evidence basis:
   - Corpus finding → cite PMIDs
   - Simulation prediction → cite binary name, parameter set, cell count
   - External literature → cite author/year
   - News source → use `[News: Author, "Title", Publication, Date. Status]` format (see `analysis/news-source-criteria.md`)
   - Expert opinion → attribute explicitly ("According to [expert]...")

5. **Absence claims require qualification.** "Not detected in the local keyword-derived analysis" is acceptable. "Does not exist in the literature" is not — our corpus is 4,830 articles, not all of PubMed. Use coverage-aware language.

6. **Chapter scope is defined by the outline.** Each chapter in `book-outline.md` has a "Non-scope" section listing what the chapter must NOT claim. Respect these boundaries. If a chapter needs to expand its scope, update the outline in the same PR.

## Citation Standards

- **PubMed:** `[PMID: XXXXX]` (existing convention, converted to `\cite{}` by generate_latex.py)
- **News sources:** `[News: Author, "Title", Publication, Date. Verified: PMID:XXXXX]` or `[Commentary: ...]` (see `analysis/news-source-criteria.md` Section 4)
- **Simulation output:** Reference the binary name and key parameters in prose (e.g., "sim-tme with O2 gradient lambda=120um, n=1M cells")
- **Figures:** `[FIGURE N: description]` — placeholder converted to `\includegraphics` by generate_latex.py

## Cross-References

Internal cross-references use section numbers: "Section 6.1" or "Chapter 8." These are plain text in v1.md (not LaTeX `\ref{}` commands). When heading numbers change, cross-references must be updated in the same PR.
