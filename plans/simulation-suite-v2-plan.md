# Ferroptosis Simulation Suite v2 — Implementation Plan

## Context

The article makes five claims that the current single-cell Monte Carlo simulation cannot validate:

1. **Spatial selectivity**: "Physical modalities restrict ROS to a defined tumor volume" — needs a spatial model
2. **Depth penetration**: "Ultrasound penetrates centimeters; light penetrates millimeters" — needs physics models
3. **ICD advantage**: "Physical modalities produce more ICD than pharmacologic inducers" — needs immune cascade
4. **Vulnerability window**: "The ferroptosis-sensitive window may last days to weeks" — needs temporal dynamics
5. **Combination therapy**: SDT + anti-PD1 optimal timing — needs treatment scheduling

No published computational model combines spatial ferroptosis with energy physics or ICD cascading. This would be a genuine first-in-field contribution.

## Simulation 1: Spatial Tumor Model (HIGHEST PRIORITY)

**Validates**: Claims 1+2 — spatial selectivity and depth penetration

### Design
- **Grid**: 500×500 cells, 20µm/cell = 1cm × 1cm tissue cross-section
- **Tumor geometry**:
  - Core (inner 60%): Glycolytic 80%, OXPHOS 15%, Persister 5%
  - Periphery (outer 40%): OXPHOS 70%, Glycolytic 20%, Persister 8%, PersisterNrf2 2%
  - 10-20 persister cluster seeds (3-5 cell radius each)
  - Border: Stromal cells (inert to ferroptosis)
- **Energy source from top** (row 0 = tissue surface):
  - **PDT**: `I(z) = I₀ × exp(-µeff × z)`, where `µeff = sqrt(3 × µa × (µa + µs'))`. At 630nm: δ ≈ 3mm
  - **SDT**: `I(z) = I₀ × 10^(-α × f × z / 10)`, α=0.7 dB/cm/MHz, f=1MHz. At 1cm: 85% intensity
  - **RSL3**: uniform concentration (systemic drug, no depth dependence)
- **Per-cell biochemistry**: Each cell runs `sim_cell_step()` with LOCAL ROS dose = base_ros × energy_at_depth
- **Bystander effect**: Dead cells release iron to 8-neighbors (Moore neighborhood). Iron diffusion D_eff ≈ 281 µm²/s
- **180 timesteps** with parallel biochemistry + sequential diffusion per step

### Key Output
- `spatial_death_{treatment}.csv` — 500×500 death maps (heatmap figures)
- `depth_kill_curves.csv` — % death vs depth for PDT/SDT/RSL3/Control
- `spatial_summary.json` — aggregate stats

### Expected Result (the paper's thesis in one figure)
- **RSL3**: Flat kill curve (uniform, but only kills ~42% of persisters)
- **PDT**: High kill at surface, exponential decay, near-zero beyond ~5mm
- **SDT**: High kill at surface, gradual decay, still ~45% intensity at 5cm
- **Control**: Uniformly near-zero

### Parallelization Strategy
1. Biochemistry step: parallelize across rows with rayon (each cell reads only its own state + immutable extra_iron)
2. Diffusion step: sequential — collect newly dead cells, distribute iron to living neighbors
3. Double-buffer pattern: "current" and "next" state arrays, swap each step

## Simulation 2: Vulnerability Window

**Validates**: Claim 4 — "ferroptosis window may be transient"

### Design
- **Recovery model**: After chemo withdrawal, persisters re-express defenses with exponential kinetics:
  ```
  fsp1(t) = fsp1_persister + (fsp1_normal - fsp1_persister) × (1 - exp(-ln2 × t / t½))
  ```
  - FSP1 t½ = 7 days, GPX4 t½ = 3 days, NRF2 t½ = 5 days, GSH t½ = 1 day
- **Timepoints**: 0h, 6h, 12h, 24h, 48h, 72h, 1wk, 2wk, 4wk
- **At each timepoint**: Generate 100K cells with recovery-adjusted params, run single-cell sim for each treatment
- **Sensitivity analysis**: Sweep recovery half-times ±50% to show robustness

### Key Output
- `vulnerability_window.csv` — death rate vs days post-chemo, per treatment
- Line chart: SDT/PDT/RSL3 sensitivity curves diverging over time

### Expected Result
- t=0: maximal vulnerability, all treatments effective
- RSL3 sensitivity drops fastest (depends on GPX4 recovery)
- SDT/PDT sensitivity persists longer (ROS burst overwhelms partial recovery)
- By 4 weeks: all approach baseline resistance

## Simulation 3: ICD-Immune Cascade

**Validates**: Claim 3 — "ICD is an advantage of physical over pharmacologic"

### Design
- **DAMP release**: Proportional to LP at death. SDT/PDT kill via runaway LP (LP >> threshold), RSL3 kills via slow LP accumulation (LP ≈ threshold). Therefore SDT/PDT produce MORE DAMP per dead cell.
- **Cascade** (deliberately simple, 3-stage linear):
  1. DAMPs activate DCs: saturating `DAMP / (DAMP + Kd)`
  2. Mature DCs prime T cells: `tcell_priming_rate × mature_dcs`
  3. T cells kill tumor cells: `tcell_kill_rate × primed_tcells × (1 - pd1_brake × (1 - anti_pd1))`
- **Comparison**: Run 100K cells per phenotype per treatment, aggregate DAMP, cascade

### Key Output
- `icd_comparison.json` — per-treatment: death_rate, total_damps, damp_per_dead_cell, immune_kills
- Bar chart: DAMP magnitude by treatment modality

### Expected Result
- SDT/PDT produce 2-5× more DAMP per dead cell than RSL3
- With anti-PD1, SDT/PDT achieve significantly more immune-mediated killing
- Qualitative ranking: SDT+anti-PD1 > PDT+anti-PD1 > RSL3+anti-PD1

## Simulation 4: Combination Therapy Optimizer

**Validates**: Claims 5+6 — combination therapy and evolutionary resistance

### Design
- **3-phase model**:
  1. **Chemo**: Kills 90% of Glycolytic/OXPHOS, spares persisters
  2. **Second-line** (at variable delay): SDT/PDT/RSL3 on surviving cells with recovery-adjusted params
  3. **Immune** cascade from ICD, with optional anti-PD1
- **Parameter sweep**: SDT delay (0-28 days) × anti-PD1 timing (concurrent, +1wk, +2wk, none)
- **Smaller grid** (200×200) for tractable sweep runtime

### Key Output
- `combo_sweep.csv` — survival fraction vs (SDT timing, anti-PD1 timing)
- `evolution_resistance.csv` — survivor phenotype distribution
- Heatmap: optimal treatment schedule

## Implementation Sequence

### Phase 1: Foundation (refactor to workspace)
1. Create workspace `Cargo.toml` and `ferroptosis-core/` crate
2. Extract `cell.rs`, `params.rs`, `biochem.rs`, `stats.rs`, `io.rs` from current `main.rs`
3. **Key refactor**: Split `sim_cell()` into `sim_cell_step()` (single timestep) + loop wrapper
4. Build `sim-original/` binary, verify **bitwise-identical** output to current `simulation_results.json`

### Phase 2: Spatial model (highest value)
5. Implement `physics.rs` with PDT/SDT/RSL3 energy models + unit tests
6. Implement `grid.rs` with tumor generation + diffusion
7. Build `sim-spatial/`, validate depth-kill curves match expected physics
8. Run full 500×500 simulation

### Phase 3: Temporal + Immune
9. Add `RecoveryRates` to params, recovery logic to `cell.rs`
10. Build `sim-window/`
11. Implement `immune.rs` (DAMP → DC → T cell cascade)
12. Build `sim-icd/`

### Phase 4: Integration + Figures
13. Build `sim-combo/` combining spatial + immune + temporal
14. Update `scripts/generate_figures.py` to render new outputs
15. Update article section 4.3 with new simulation results

|------|--------|
| Bitwise reproduction | `sim-original` output == current `simulation_results.json` |
| PDT physics | `pdt_intensity_at_depth(3000µm)` ≈ 1/e at δ=3mm |
