## Part 3: Directory Structure

### Design Principles

1. **Claude-searchable** вЂ” Glob patterns and Grep work instantly across the entire corpus
2. **Deterministic paths** вЂ” Given a PMID or DOI, you can compute the exact file path
3. **Flat where possible** вЂ” Avoid deep nesting; Claude reads file lists faster
4. **Metadata as frontmatter** вЂ” Every article file has YAML frontmatter for structured search
5. **Index files** вЂ” Machine-readable indexes for fast lookup without reading every file

### Article File Format (corpus/by-pmid/XXXXXXXX.md)

Each article is stored as a markdown file with YAML frontmatter. This lets Claude `Grep` for any field and `Read` the full content.

```markdown
---
pmid: "38000001"
doi: "10.1038/s41586-024-07000-1"
title: "Tumor-treating fields combined with immune checkpoint inhibitors..."
authors: ["Smith J", "Lee K", "Wang L"]
journal: "Nature Cancer"
year: 2024
month: 3
volume: "5"
issue: "3"
pages: "412-428"
impact_factor: 23
access: "open"
mechanisms: ["ttfields", "immunotherapy"]
cancer_types: ["glioblastoma"]
evidence_level: "phase2-clinical"
mesh_terms: ["Tumor Treating Fields", "Immune Checkpoint Inhibitors", "Glioblastoma"]
genes: ["PD-L1", "CTLA4"]
drugs: ["pembrolizumab", "Optune"]
citations_count: 47
icite_rcr: 8.2
pubtator_annotations: true
date_added: "2026-03-28"
---

## Abstract

[Full abstract text here]

## Key Findings

- [Extracted key findings, 3-5 bullet points]

## Full Text

[Full article text if available, otherwise "Abstract only вЂ” full text paywalled"]

## Notes

[Any manual annotations added during review]
```

### INDEX.jsonl Format

One JSON object per line for fast programmatic lookups:

```json
{"pmid":"38000001","doi":"10.1038/s41586-024-07000-1","title":"Tumor-treating fields combined with...","journal":"Nature Cancer","year":2024,"mechanisms":["ttfields","immunotherapy"],"cancer_types":["glioblastoma"],"evidence_level":"phase2-clinical","access":"open"}
```

### How Claude Searches This Structure

| Task | Command |
|------|---------|
| Find all TTFields articles | `Grep "ttfields" tags/by-mechanism/ttfields.txt` or `Grep 'mechanisms:.*ttfields' corpus/by-pmid/` |
| Find all glioblastoma research | `Grep "glioblastoma" tags/by-cancer-type/glioblastoma.txt` |
| Find articles combining two mechanisms | `Grep 'mechanisms:.*immunotherapy.*ttfields' corpus/by-pmid/` |
| Read a specific article | `Read corpus/by-pmid/38000001.md` |
| Find all Phase 3 clinical trials | `Grep "phase3-clinical" tags/by-evidence-level/phase3-clinical.txt` |
| Find articles mentioning a gene | `Grep "BRAF" corpus/by-pmid/` |
| Count articles by mechanism | `wc -l tags/by-mechanism/*.txt` |
| Find recent additions | `Grep "date_added: \"2026-03" corpus/by-pmid/` |
| Full-text search for a concept | `Grep "Warburg effect" corpus/by-pmid/` |
| List all articles from a journal | `Read tags/by-journal/nature-cancer.txt` |

### Why This Structure Works for Claude

1. **YAML frontmatter** = structured fields searchable via Grep regex patterns
2. **Tag index files** = pre-computed lists, instant lookup without scanning all articles
3. **PMID-keyed flat files** = deterministic path, no ambiguity, Glob `corpus/by-pmid/*.md` lists everything
4. **JSONL index** = machine-parseable master list, one `Read` gives full corpus overview
5. **Markdown format** = Claude reads it natively, no binary parsing needed
6. **No deep nesting** = Grep/Glob patterns stay simple and fast

### Key Corrections to Our Pipeline

1. **Unpaywall**: Must use real email, not `@example.com`. Some DOIs return 404. **OpenAlex includes Unpaywall OA data** and is more reliable вЂ” use OpenAlex as primary OA source.
2. **Semantic Scholar**: Aggressively rate-limits. **Must get free API key** before building fetch scripts.
3. **CORE API**: **Requires registration** for free API key. Cannot use anonymously.
4. **bioRxiv API**: Date-range listing only, NOT a search API. For keyword search of preprints, use Europe PMC or OpenAlex with source filter.
5. **CrossRef**: No OA status вЂ” pair with OpenAlex for OA determination.
